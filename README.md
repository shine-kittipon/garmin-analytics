# garmin-analytics

Personal running analytics platform powered by Garmin (Forerunner 170) data and an AI coach.

**Goal:** Complete a 10 km run in under 60 minutes.

## What this is

- **Daily sync** via GitHub Actions (01:00 UTC) — runs while PC is off
- **Multi-table SQLite database** stored in the repo for local power-querying
- **CSV + JSON exports** per table — paste a raw GitHub URL to any AI for instant analysis
- **Weekly coaching report** generated by Claude (committed every Monday to `data/reports/`)
- **MCP server** for on-demand AI coaching via Claude Desktop / Claude Code

## Architecture

```
.github/workflows/sync.yml          cron daily 01:00 UTC
  └─ garmin_sync/sync.py            auth → fetch → transform → upsert → export
       ├─ garmin_sync/auth.py       garth token login (MFA-safe)
       ├─ garmin_sync/source.py     GarminSource protocol + GarminConnectSource impl
       ├─ garmin_sync/endpoints.py  per-domain fetch functions
       ├─ garmin_sync/transform.py  raw JSON → normalized rows
       ├─ garmin_sync/store.py      SQLite upsert + CSV/JSON export
       └─ garmin_sync/schema.sql    table definitions
  └─ coach/report.py                weekly markdown report (Mondays, via Anthropic API)
       └─ coach/features.py         CoachSummary: load, pace trend, sub-60 projection

coach/mcp_server.py                 on-demand AI coaching via MCP tools
data/
  db/garmin.sqlite                  SQLite — relational queries
  csv/*.csv                         text export — paste raw URL to any AI
  json/*.json                       text export — same
  reports/weekly-YYYY-Www.md        weekly coaching reports
```

## Data collected

| Domain | Tables | Key metrics |
|---|---|---|
| Running activities | `activities`, `activity_laps` | pace, HR zones, cadence, VO2max, training load |
| Daily health | `daily_summary` | steps, calories, resting HR, stress, body battery, SpO2 |
| Sleep | `sleep` | duration, stages, sleep score |
| HRV | `hrv` | last night avg, weekly avg, status |
| Training | `training` | VO2max, training status, acute/chronic load, readiness |
| Personal records | `personal_records` | PRs by type |

## Quick start

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/garmin-analytics.git
cd garmin-analytics
pip install garminconnect garth pandas anthropic mcp
```

### 2. Generate a Garmin token (one time, on your PC)

```python
from garminconnect import Garmin
c = Garmin("you@example.com", "yourpassword")
c.login()
print(c.garth.dumps())   # copy this output
```

### 3. Add GitHub Actions Secrets

In your repo: Settings → Secrets and variables → Actions → New repository secret:

| Secret | Value |
|---|---|
| `GARMIN_TOKEN` | output of `c.garth.dumps()` above |
| `ANTHROPIC_API_KEY` | your Anthropic API key (for weekly reports) |

`GARMIN_EMAIL` + `GARMIN_PASSWORD` are optional fallbacks if you don't use MFA.

### 4. Run a local backfill

```bash
export GARMIN_TOKEN="your-token-here"
python -m garmin_sync.sync --start 2026-01-01 --end 2026-06-28
```

### 5. Enable the MCP server (Claude Desktop / Claude Code)

Add to your Claude Desktop `claude_desktop_config.json`:
```json
{
  "mcpServers": {
    "garmin-analytics": {
      "command": "python",
      "args": ["-m", "coach.mcp_server"],
      "cwd": "C:/Users/kitti/garmin-analytics"
    }
  }
}
```

Then ask Claude: *"Am I improving toward my sub-60 10K goal?"*

## Local analysis with DuckDB

```python
import duckdb
duckdb.sql("SELECT date, pace_min_km, avg_hr FROM activities ORDER BY date DESC LIMIT 10").show()
```

Or via pandas + SQLite directly:
```python
import sqlite3, pandas as pd
conn = sqlite3.connect("data/db/garmin.sqlite")
df = pd.read_sql("SELECT * FROM activities ORDER BY date DESC", conn)
```

## AI access without your PC

When your PC is off and you want to ask an AI about your data, paste a raw GitHub URL:

```
https://raw.githubusercontent.com/YOUR_USERNAME/garmin-analytics/main/data/csv/activities.csv
```

Any AI (ChatGPT, Claude web) can read CSV/JSON text directly from the URL.

## Notes

- Uses the unofficial `garminconnect`/`garth` library — the official Garmin Developer API
  is restricted to registered businesses and is currently suspended for new applicants.
- The official Training API only *pushes* workouts to devices; it does not return
  training load, VO2max, or readiness data.
- Garmin Forerunner 170 does not support all metrics (e.g. running power, advanced HRV).
  Missing values are stored as `NULL` and rendered as "—" in reports.
